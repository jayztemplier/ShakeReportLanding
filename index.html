Shake Report

beta

an easy to use, open source, bug reporter for mobile applications.

Shake Reporter is a framework that allows you to easily get bug reports from users of your mobile application. When you see something wrong, shake the device, and report the issue with a custom message. You can either receive reports by email or configure a shake report backend which will collect and classify the reports.

How does it work?

Shake Report record a lot of information while the tester is trying the application, and doesn't require any action to activate it. During a session, we:

  • record the screen, and securely hide secure field
  • display all user's gestures on the video
  • take a screenshot of the last screen viewed by the user, and save the dumped view
  • save the logs of the session, and crash reports

Installation

Download the iOS SDK on github

Copy/Past the library folder in your XCode project

Import the frameworks: QuartzCore, MessageUI

Initialize the reporter in your Application Delegate:

SRReporter *reporter = [SRReporter reporter];
[reporter setDefaultEmailAddress:@"example@shakereporter.com"];
[reporter startListener];

You're all setup!

Usage

Shake the device... that's it!

With a backend

Description

The Shake Reporter backend has been written in Ruby, using Ruby on Rails. By default, it uses Sendgrid (for the production environment). Originally, it's been written to be super easy to install on Heroku. The installation process explains how to deploy your own instance of the backend on Heroku.

Deploy on Heroku

Clone the github repository on your computer.

Open a terminal

cd project_directory

Create a new heroku app

heroku create my_app_name

Enable SendGrid, Scheduler, and MongoHQ for your heroku app

heroku addons:add mongohq
heroku addons:add scheduler
heroku addons:add sendgrid:starter

Setup a user

heroku config:set SR_USERNAME=myusername
heroku config:set SR_PASSWORD=mypassword

Commit and deploy the code

git add .
git commit -a -m "my first commit"
git push heroku master

Now your application should be deployed, and accessible at my_app_name.herokuapp.com

iOS installation

Follow the instructions on the "By email" side (column on the left)

Replace the code on the Application Delegate by:

SRReporter *reporter = [SRReporter reporter];
NSURL *url = [NSURL URLWithString:@"http://my_app_name.herokuapp.com/reports.json"];
[reporter startListenerConnectedToBackendURL:url];

You now have to specify the credentials you set up for the backend

[reporter setUsername:@"myusername"];
[reporter setPassword:@"mypassword"];

You are ready to go!

Usage

Shake the device... a form will appear. You can specify a title and add more information about the context and how to reproduce the issue. A confirmation message will appear when the report has been receive by the server.

Go on the website, a new report should have been created as an Open report. You can open it to have access to a lot of information such as screenshots, dumped view, logs, last crash report or custom information that you set up with the iOS SDK.
